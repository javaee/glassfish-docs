To Grant Access to Administration Tools

Only users in the asadmin group are able to access the Administration Console and the asadmin command line utility. To give users access to these administration tools, add them to the asadmin group in the admin-realm.

  1. In the navigation tree, expand the Configuration node.
  2. Expand the Security node.
  3. Expand the Realms node.
  4. Select the admin-realm node.

    The Edit Realm page opens.

  5. On the Edit Realm page, click the Manage Users button.

    The File Users page opens.

    Initially after installation, the administrator user name and password entered during installation are listed in a file named admin-keyfile. By default, this user belongs to the group asadmin, which gives rights to modify the Enterprise Server. Assign users to this group only if you want to grant them administrator privileges for the Enterprise Server.

    If you add users to the admin-realm realm, but assign the user to a group other than asadmin, the user information will still be written to the file named admin-keyfile, but the user will not have access to administrative tools or to applications in the file realm.

  6. On the File Users page, click New to add a new user to the admin-realm realm.

    The New File Realm User page opens.

  7. In the User ID field, type the new user name.
  8. In the Group List field, type the names of the groups to which the user will belong.

    To authorize a user to make modifications to the Enterprise Server, include the asadmin group in the Group List.

  9. In the New Password and Confirm New Password fields, type the password for the new user.
  10. Click OK to add this user to the admin-realm realm, or click Cancel to quit without saving.
